My macOS/iOS setup (with config) is described [here](/config).

### Websites

- [Blade](https://blade.im) for all websites with [React](https://react.dev)
- [Jazz](https://jazz.tools) for when I need collabortive state & [Reatom](https://v1000.reatom.dev) for more complex client interactions
- Deployed on [Vercel](https://vercel.com) or [Cloudflare](https://cloudflare.com)
- [Panda CSS](https://panda-css.com) or [Tailwind](https://tailwindcss.com) for styling

### HTTP

- TS: [Hono](https://hono.dev) (distributed with Cloudflare) or [Encore](https://encore.dev) (single origin on GCP) depending on task. [Elysia](https://elysiajs.com) is nice too.
- Go: [Encore](https://encore.dev)
- Rust: [Axum](https://github.com/tokio-rs/axum)

### CLIs

- Go with [go-snap](https://github.com/dzonerzy/go-snap)
- TS with [Bun](https://bun.sh) & [Optique](https://github.com/dahlia/optique)
- Rust with [clap](https://github.com/clap-rs/clap)

### iOS

- [Expo](https://expo.dev) + Expo plugins
- [Unistyles](https://unistyl.es) for styles
- Usually talking to Hono endpoints that serve [Blade](https://blade.im). Using [Jazz](https://jazz.tools) & [Reatom](https://v1000.reatom.dev) where appropriate

### Desktop

- Wrap websites with [Tauri](https://tauri.app) for desktop apps

### Infra

- Manage services with [Alchemy](https://alchemy.run), deployed often on AWS or Cloudflare

### Analytics

- [Openpanel](https://github.com/Openpanel-dev/openpanel) for analytics (site visits etc.)
- [Sentry](https://sentry.io) for errors
- [Axiom](https://axiom.co) for logs
